In English

Demo Project

It is important to understand that this is a "demo project". Simply a demo project showcasing my skills in Django development. Additionally, it served as a playground for me to dive deeper into the framework and learn how to do things I hadn't done before or only partially understood.

Technologies

This project demonstrates proficiency in:

  • Python
  • Django
  • PostgreSQL (using Django ORM, although I'm also familiar with raw SQL)
  • Nginx
  • Docker
  • Gunicorn
  • Basic HTML/CSS/JavaScript

In my experience, I have encountered many programmers who didn't fully grasp the capabilities of their framework and ended up implementing functionalities already provided by the framework. First and foremost, I aim to avoid such an approach in my work.

Concept

This blog appears as a "simple blog," and that's exactly what it should be. I focused on a minimal set of features (though I implemented them in interesting ways for my personal development), ensuring they work flawlessly and are polished. My goal was to create a simple and understandable project for my portfolio, showcasing quality work rather than a commercial project that I would continue to develop.

However, this blog implementation significantly differs from the blogs created in development courses. The main difference lies in the fact that you can independently add a publication, and it will be displayed.

Implementation

The majority of the functionality was implemented as comprehensively as possible for a demo project:

  • The main feature is the ability for users to add content. I used the most "user-friendly" editor I could find while protecting the project from spam by limiting post submissions to once every 10 minutes. Editing existing posts has no restrictions.
  • The same editor was finely tuned and customized to match my vision. I even had to fix some plugin developer's mistakes.
  • Implemented security measures, including spam protection (limiting publications within a specified time) and preventing the insertion of unwanted code using a preprocessor for the text editor.
  • Replaced Django's default username authentication with email authentication (while preserving username authentication in the admin panel because how can we live without admin:admin).
  • Added a unique email requirement to the corresponding field in the database.
  • Configured clear error messages for authentication failures, explaining possible causes to the user in simple and understandable language.
  • Implemented proper editing of user data using two forms on a single page (which was a bit challenging).
  • Configured an accurate user activity tracker (signals.py).
  • Naturally, the admin panel was also configured.
  • Time display (for comments and posts) is adjusted according to your time zone (middleware.py).
  • Implemented middleware to ensure correct redirection from pages ending with a slash to slash-less pages ("address/" → "address") to avoid content duplication.
  • Implemented functionality for users to manage their own publications.
  • The layout is responsive, adapting to desktops, tablets, and smartphones. Additionally, I independently designed a stylish layout for user article tags to seamlessly integrate them into the site's design.
  • Some elements required my own "redesigning" and rearranging to create a more user-friendly UX/UI.
  • The code is thoroughly documented, describing the functionality of classes, functions, and methods.
  • Everything was packaged into a Docker container with configurations for gunicorn and Nginx.

I cannot say how well-suited this implementation is for your specific company/startup, but based on my previous experience managing a development department, my junior developers would not have been able to handle a project of this caliber. This would be the level of a competent middle developer in our company.

На русском

Демонстрационный проект

Важно понимать, что это "демонстрационный проект". Просто демонстрационный проект, который показывает мои навыки в области разработки на Django. Кроме того, это был полигон для меня, чтобы углубиться в фреймворк и научиться делать вещи, которые я ранее не делал или делал не до конца понимая, как они работают.

Технологии

Данный проект подтверждает умение работать с:

  • Python
  • Django
  • PostgreSQL (используя Django ORM, хотя я также знаком с чистым SQL)
  • Nginx
  • Docker
  • Gunicorn
  • И базовые знания HTML/CSS/JavaScript

В своей практике я встречал множество программистов, которые не до конца понимали все возможности своего фреймворка и самостоятельно реализовывали функции, которые уже предусмотрены в фреймворке. В первую очередь, я хочу избегать подобного подхода в своей работе.

Концепция

Этот блог выглядит как "просто блог", и таким он и должен быть. Здесь я сконцентрировался на минимальном количестве функций (хотя и реализовал их интересным образом, для моего личного развития), но при этом все они должны работать безотказно и быть отполированными. Моя задача заключалась в создании простого и понятного проекта для портфолио, который демонстрирует качественную работу, а не коммерческий проект, который я потом развивал бы.

Однако, данная реализация блога сильно отличается от тех блогов, что делают на курсах по разработке. Основное отличие заключается в том, что вы можете самостоятельно добавить публикацию и она будет отображаться.

Реализация

Большая часть функционала сделана максимально комплексно, насколько это было возможно для демонстрационного проекта:

  • Главная функция - возможность пользователей добавлять контент. Я использовал самый "симпатичный" редактор, который смог найти, и при этом защитил проект от спама путем ограничения добавления постов раз в 10 минут. Редактирование уже существующих постов не имеет ограничений;
  • Тот же редактор был тонко настроен и доработан под мое виденье редактора. Пришлось даже ошибки разработчиков плагина исправлять;
  • Реализованы меры безопасности, включая защиту от спама (ограничение на публикацию в течение определенного времени) и предотвращение вставки нежелательного кода с использованием препроцессора для текстового редактора;
  • Стандартная аутентификация Django по username изменена на аутентификацию по email (причем в админке сохранена аутентификация по username, как же мы будем жить без admin:admin);
  • Добавлено условие уникальности email в соответствующем поле в базы данных;
  • Настроены понятные сообщения об ошибках аутентификации и их возможные причины, объясненные пользователю простым и понятным языком;
  • Реализовано корректное редактирование пользовательских данных с использованием двух форм на одной странице (это было немного трудно);
  • Настроен корректный трекер активностей пользователя (signals.py);
  • Админку, разумеется, тоже настроил;
  • Вывод времени (комментариев и постов) настроен в соответствии с вашим часовым поясом (middleware.py);
  • С помощью промежуточного программного обеспечения (middleware), была настроена корректная работа перенаправлений со страницами, оканчивающимися на слэш, на страницы без слэша ("address/" → "address"), чтобы избежать дублирования контента.
  • Реализован функционал управления своими публикациями пользователя;
  • Верстка адаптирована под десктопы, планшеты и смартфоны. Кроме того, я самостоятельно разработал стильную верстку для тегов пользовательских статей, чтобы они органично вписывались в дизайн сайта.;
  • Часть элементов пришлось самому “редизайнить” и переставлять, чтобы UX/UI сделать дружелюбнее;
  • Код максимально задокументирован. Вся функциональность классов, функций и методов описана;
  • Все это было упаковано в Docker-контейнер, сконфигурированы gunicorn и Nginx.

Не знаю насколько это "компетентная" реализация конкретно для Вашей компании/стартапа, но в моем предыдущем опыте управления отделом разработки, мои junior-разработчики с таким бы не справились. Это был уровень хорошего middle-разработчика в нашей компании.
